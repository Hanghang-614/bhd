# Security Policy

## Reporting a Vulnerability

Please do not open a public issue for vulnerabilities or leaked sensitive data.

Use GitHub private vulnerability reporting if it is enabled for this repository. If it is not enabled, contact the repository owner through GitHub and share only the minimum information needed to start a private conversation.

Useful report details include:

- A short description of the issue.
- Steps to reproduce.
- The affected command, API route, parser, or storage path.
- Whether secrets, personal data, or local files can be exposed.

## Sensitive Data

BHD Memory is designed to ingest personal transcripts and knowledge files, so contributors should treat test fixtures and examples carefully. Do not commit real API keys, private transcripts, production databases, customer data, or local `.bhd/` contents.
